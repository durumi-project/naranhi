# 학폭 판례 자동 수집 스크립트

법제처 OPEN API로 학교폭력 관련 판례를 한 번에 수집해서, 우리 `cases.json` 스키마에 맞춰 정리해주는 Python 스크립트입니다.

## 빠른 시작 (3분)

### 1. Python 3.8 이상이 설치되어 있는지 확인

터미널 또는 명령 프롬프트를 열고:

```bash
python3 --version
```

`Python 3.8.x` 이상이 보이면 OK. 안 보이면 [python.org](https://www.python.org/downloads/)에서 설치.

> 윈도우는 `python --version`, 맥은 `python3 --version`이에요. 헷갈리면 둘 다 시도해보세요.

### 2. 스크립트 다운로드

`collect_cases.py` 파일을 작업할 폴더에 저장하세요. 예를 들어 `~/Documents/naranhi/` 같은 곳에.

### 3. 실행

같은 폴더에서 터미널을 열고:

```bash
python3 collect_cases.py
```

OC가 `durumitech`로 미리 설정되어 있어서 그냥 실행하면 작동합니다. (다른 OC를 쓰려면 다음 섹션 참조)

### 4. 결과 확인

실행이 끝나면 `output_collected/` 폴더가 생기고 그 안에:

- **학폭_판례_원본.csv** — 변호사 검수용 (엑셀로 열기)
- **학폭_판례_원본.json** — 학생팀 친화 변환 작업용
- **수집_리포트.md** — 어떤 키워드에서 몇 건씩 모였는지

## 다른 OC를 쓰고 싶다면

### 방법 A · 파일에서 직접 수정

`collect_cases.py` 파일을 메모장/VS Code 등으로 열고 다음 줄을 찾아 변경:

```python
LAW_OC = os.environ.get("LAW_OC", "durumitech")
                                    # ^^^^^^^^^^ 여기를 본인 OC로
```

### 방법 B · 환경변수 (안전)

```bash
# 맥/리눅스
LAW_OC=my_email_id python3 collect_cases.py

# 윈도우 명령 프롬프트
set LAW_OC=my_email_id
python collect_cases.py

# 윈도우 PowerShell
$env:LAW_OC = "my_email_id"
python collect_cases.py
```

방법 B는 코드를 GitHub에 올릴 때 OC가 노출되지 않아 안전해요.

## 무엇을 가져오나요?

스크립트는 8개 키워드로 동시 검색해서 *중복 제거*한 후, 각 판례의 **본문**까지 가져와요.

8개 키워드:
- `학교폭력` — 가장 일반적
- `학교폭력예방법` — 법률 명시 사건
- `학교폭력대책심의위원회` — 학폭위 관련
- `정보통신망 명예훼손 학생` — 사이버 명예훼손
- `사이버 학교폭력` — 사이버 일반
- `촉법소년` — 소년보호사건
- `학생 폭행` — 형사 폭행
- `단톡방 모욕` — 단톡방 사건

각 판례는 우리 `cases.json` 스키마에 매핑되는데:

### 자동으로 채워지는 필드 (✅)

| 필드 | 출처 |
|---|---|
| `case_id` | `AUTO-` + 판례일련번호 |
| `decision_date` | 선고일자 |
| `court` | 법원명 |
| `case_number` | 사건번호 |
| `case_title_formal` | 사건명 |
| `disposition_summary` | 판결유형 + 선고 |
| `original_summary` | 판시사항 |
| `original_facts_raw` | 판결요지 |
| `original_law` | 참조조문 |
| `original_full_text` | 판례내용 본문 |
| `keywords_auto` | 본문에서 도메인 키워드 자동 매칭 |
| `_category_auto` | 행정/형사/민사/소년보호 자동 분류 |

## 자주 발생하는 문제

### "HTTP 오류 403: Forbidden"이 떠요

OC가 잘못됐거나 신청한 API가 아니에요. 확인할 것:
1. `LAW_OC` 변수가 정확한 이메일 ID인지 (예: `durumitech@gmail.com` → `durumitech`)
2. 법제처 OPEN API 신청에서 *판례* 항목을 체크했는지
3. 신청이 *승인 완료* 상태인지 (마이페이지 → 신청 내역)

### 0건만 모이고 끝나요

OC가 작동하지 않거나 네트워크 문제. 브라우저에서 다음 URL을 열어 직접 확인해보세요:

```
https://www.law.go.kr/DRF/lawSearch.do?OC=durumitech&target=prec&type=XML&query=학교폭력&display=10
```

XML이 보이면 OC는 정상. 안 보이면 OC를 본인 것으로 바꿔보세요.

### 한 키워드가 오래 걸려요

페이지당 100건씩 3페이지(최대 300건)까지 가져와요. 키워드에 결과가 많으면 시간이 더 걸려요. `MAX_PAGES_PER_KEYWORD = 1` 로 줄이면 빠르게 테스트만 가능.

### 결과가 너무 적어요

스크립트 상단의 `KEYWORDS` 리스트에 키워드를 추가하세요. 예:

```python
KEYWORDS = [
    "학교폭력",
    # ... 기존 8개
    "학생 명예훼손",   # 추가
    "학생 협박",       # 추가
    "성희롱 학교",     # 추가
]
```

### 본문이 비어 있는 판례가 있어요

일부 판례는 API에서 본문(`판례내용`)을 제공하지 않아요. 보통 너무 오래된 사건이거나 *기밀*에 해당하는 경우. 메타 정보만 활용하거나 해당 건은 건너뛰면 돼요.

## 알아두면 좋은 한계점

1. **법원 판결문만 모입니다.** 학폭위 결정·교육청 학폭 사례·두루 자료는 이 API에 없어서 수동 수집이 필요해요.
2. **사건 유형이 한쪽으로 치우칠 수 있어요.** 키워드 "학교폭력"만 쓰면 *행정 사건(처분 불복)*이 많이 나와요. 그래서 8개 키워드를 다양하게 섞은 거예요.
3. **친화 변환은 자동화 불가능.** 친화_변환_가이드_v1.pdf의 5대 원칙은 사람이 적용해야 해요.
4. **두루 변호사 검수도 자동화 불가.** 판례_검수_체크리스트_v1.pdf를 활용해 검수.
